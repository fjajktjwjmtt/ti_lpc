Texas Instruments LPC Voice Synthesizer Decoder (e.g: Speak and Spell) v1.1.5
Copyright (C) 2019-2026 BrerDawg, et. al.

Minimal parameter checks are performed, params are delimited by spaces,
therefore, do not use spaces in filenames or strings on the command line.

Anything before a colon is ignored when strings are rendered, this allows a
convenient name to be at head of each string.

Some commands use default values if none are specified.

Pick the chip def file which sounds best.

A hex string file is generated after render of audio, named: 'zzzlast_utter_strhex.txt'

Ubuntu users could add this to end of command to hear audio: && aplay zzzout.wav


Parameters:
-----------

  mode=         Operating mode. Required. Values:
                  render           - render LPC data to audio file
                  romlist          - dump list of words found in rom
                  rendaddrfileline - render rom addr from file at line index
                  rendaddrfileseq  - render rom addr from file sequentially
                  rendstrfileline  - render hex string from file at line index
                  rendstrfileseq   - render hex string from file sequentially
                  cleanbrace       - clean file keeping only content within braces
                  cleanquote       - clean file keeping only content within quotes

  chip=         Chip definition file (e.g: tms5100.txt, tms5110.txt, tms5200.txt, tms5220.txt).
                If compiled with USE_TMS5K_H, a chip name without extension can be used
                to load compiled-in params (e.g: chip=tms5220).

  LPC data input (use only one at a time, mutually exclusive):
  str=          Decimal comma-separated LPC byte string (e.g: str=45,36,126,14,3,255)
                Values must be 0-255. Input is validated for invalid characters.
  strhex=       Hex comma-separated LPC byte string (e.g: strhex=A5,4F,7A,D3)
                Supports optional 0x prefix. Input is validated for invalid characters.
  strfile=      Text file containing decimal comma-separated LPC bytes (e.g: strfile=data.txt)
                File content is the same format as str= (e.g: 165,79,122,211,60,90)
                File is read, validated, and rendered as decimal data.
  strhexfile=   Text file containing hex comma-separated LPC bytes (e.g: strhexfile=data_hex.txt)
                File content is the same format as strhex= (e.g: A5,4F,7A,D3 or 0xA5,0x4F)
                Trailing commas are acceptable. File max size is 1MB.
  strbin=       Binary LPC file path (e.g: strbin=0220_Affirmative.lpc)
                File is read as raw bytes and passed directly to the LPC decoder.
  addr=         ROM hex address to render (requires rom0=)

  rom0=         Primary VSM ROM file
  rom1=         Secondary VSM ROM file (requires rom0=)

  wav=          Output audio filename (default: zzzout.wav).
                Format is auto-detected from extension: .wav, .au, .aiff, .raw
  srate=        Output sample rate in Hz (default: 8000, range: 4000-192000)
  swidth=       Output sample width in bits (default: 16, values: 8 or 16)
  output=       Output channel mode (default: st). Values:
                  st or stereo  - stereo output (2 channels)
                  mo or mono    - mono output (1 channel)
  ch=           Audio channel placement for stereo output (default: both). Values:
                  left, l, 0    - audio on left channel, silence on right
                  right, r, 1   - audio on right channel, silence on left
                If not specified with stereo output, audio goes to both channels.
                Ignored for mono output.

  gain=         Audio gain percentage (default: 75, range: 0-300)
  filt=         Lattice filter (default: on). Set filt=off to disable.
  verb=         Verbose output. Set verb=on to enable.

  fnamein=      Input filename for file-based modes
  fnameout=     Output filename for romlist mode (default: zzzaddr_list.txt)
  line=         Zero-based line index for file-based render modes
  step=         Step direction/amount for sequential file-based modes (default: 1)


Invocation Examples:
--------------------

ti_lpc_cmd -help
ti_lpc_cmd --help

--- ROM listing ---

ti_lpc_cmd mode=romlist rom0=tmc0351n2l.vsm rom1=tmc0352n2l.vsm fnameout=zzzaddr_list.txt
    //dump list of words found in rom to text file, formatted as 'address' 'word' (e.g: 6a8d isle)

ti_lpc_cmd mode=romlist rom0=tmc0351n2l.vsm rom1=tmc0352n2l.vsm fnameout=zzzaddr_list.txt verb=on
    //with verbose enabled for detailed output

--- Render from ROM address ---

ti_lpc_cmd mode=render chip=tms5100.txt rom0=tmc0351n2l.vsm rom1=tmc0352n2l.vsm wav=zzzout.wav gain=75 addr=12a1
    //render rom hex address 12a1 with gain of 75%

ti_lpc_cmd mode=render chip=tms5100.txt rom0=tmc0351n2l.vsm rom1=tmc0352n2l.vsm addr=1600
    //render rom hex address 1600 to default 'zzzout.wav'

--- Render from binary LPC file ---

ti_lpc_cmd mode=render chip=tms5220.txt wav=output.wav strbin=0220_Affirmative.lpc
    //render binary LPC file to stereo 16-bit 8kHz wav (default settings)

ti_lpc_cmd mode=render chip=tms5220.txt wav=output.wav strbin=0220_Affirmative.lpc output=mono
    //render to mono wav

ti_lpc_cmd mode=render chip=tms5220.txt wav=output.wav strbin=0220_Affirmative.lpc ch=left
    //render to stereo with audio on left channel only, right channel silence

ti_lpc_cmd mode=render chip=tms5220.txt wav=output.wav strbin=0220_Affirmative.lpc ch=right
    //render to stereo with audio on right channel only, left channel silence

ti_lpc_cmd mode=render chip=tms5220.txt wav=output.wav strbin=0220_Affirmative.lpc srate=48000
    //render upsampled to 48kHz stereo

ti_lpc_cmd mode=render chip=tms5220.txt wav=output.wav strbin=0220_Affirmative.lpc output=mono swidth=8
    //render to 8-bit mono wav

--- Render from decimal string ---

ti_lpc_cmd mode=render chip=tms5200.txt wav=zzzout.wav str=45,36,126,14,3,255...7,40
    //render decimal string to audio file (no roms required, strings must not have spaces)

ti_lpc_cmd mode=render chip=tms5100.txt str=isle:45,36,126,14,3,255...12,255
    //render decimal string (str has word label prefixed before colon, label is ignored)

--- Render from hex string ---

ti_lpc_cmd mode=render chip=tms5220.txt strhex=45,36,AE,D5,0x56,A7...,AE,D5
    //render hex string to audio file

ti_lpc_cmd mode=render chip=tms5220.txt filt=off strhex=45,36,AE,D5,0x56,A7...03,DF
    //render hex string without lattice filtering

ti_lpc_cmd mode=render chip=tms5100.txt strhex=4D,E6,C8,C2,91,8F,BB,9C,72,89,98,84,35,7D,D5,12,D6,98,5A,FA,E2,C1,94,71,0F,B7,D9,8D,09,9F,61,66,B7,13,E3,D3,4D,BC,1F,05,46,9E,AB,65,AD,0E,86,5E,53,F5,2A,EB,BD,CB,F4,B8,9B,96,14,9F,4B,52,2D,C4,B5,2B,CF,74,8A,8B,62,11,1D,EB,D4,E4,CC,34,6B,D2,69,C5,85,9B,D6,A5,3B,CF,8E,BE,4C,E5,A6,CE,9B,6F,78,C9,6B,08,94,FB,FE,F2,2B,CF,D9,54,F7,B4,F4,00

--- Render from text file (decimal) ---

ti_lpc_cmd mode=render chip=tms5220.txt wav=output.wav strfile=lpc_decimal.txt
    //render LPC data from a text file containing comma-separated decimal values (0-255)
    //file content example: 165,79,122,211,60,90,143,174,200,169

--- Render from text file (hex) ---

ti_lpc_cmd mode=render chip=tms5220.txt wav=output.wav strhexfile=lpc_hex.txt
    //render LPC data from a text file containing comma-separated hex values
    //file content example: A5,4F,7A,D3,3C,5A,8F,AE,C8,A9
    //also accepts 0x prefix: 0xA5,0x4F,0x7A,0xD3
    //trailing commas are acceptable: A5,4F,7A,D3,

--- File cleanup ---

ti_lpc_cmd mode=cleanbrace fnamein=Vocab_US_Large.cpp fnameout=Vocab_US_Large_str_clean.txt
    //clean a file keeping only what's within braces (useful for extracting strings from C code)

ti_lpc_cmd mode=cleanquote fnamein=a_quote_hex_string_file.txt fnameout=a_hex_string_file.txt
    //clean a file keeping only what's within quotes

--- Sequential file-based rendering (ROM addresses) ---

ti_lpc_cmd mode=rendaddrfileseq chip=tms5100.txt rom0=tmc0351n2l.vsm rom1=tmc0352n2l.vsm fnamein=zzzaddr_list.txt line=0
    //render rom addr from file at line 0 (stores next index in 'zzzline_index.txt')

ti_lpc_cmd mode=rendaddrfileseq chip=tms5100.txt rom0=tmc0351n2l.vsm rom1=tmc0352n2l.vsm fnamein=zzzaddr_list.txt
    //render next rom addr using index from 'zzzline_index.txt', auto-increments

ti_lpc_cmd mode=rendaddrfileseq chip=tms5100.txt rom0=tmc0351n2l.vsm rom1=tmc0352n2l.vsm fnamein=zzzaddr_list.txt step=-2
    //step backward by 2 lines

--- Sequential file-based rendering (hex strings) ---

ti_lpc_cmd mode=rendstrfileseq chip=tms5220.txt fnamein=Vocab_US_Large_str_clean.txt line=0
    //render hex string from file at line 0

ti_lpc_cmd mode=rendstrfileseq chip=tms5220.txt fnamein=Vocab_US_Large_str_clean.txt
    //render next hex string, auto-increments

ti_lpc_cmd mode=rendstrfileseq chip=tms5220.txt fnamein=Vocab_US_Large_str_clean.txt step=-3
    //step backward by 3 lines
